## Why unextension?

<CardGrid stagger>
  <Card title="Write once" icon="approve-check">
    Build your plugin UI as a standard web app with any framework — React, Vue, Svelte, or plain
    HTML.
  </Card>
  <Card title="Deploy everywhere" icon="rocket">
    Generate a VS Code extension and a JetBrains plugin from the same codebase with a single
    command.
  </Card>
  <Card title="Universal bridge" icon="puzzle">
    Use the `@unextension/bridge` API to interact with the editor — file system, workspace, UI — in
    a target-agnostic way.
  </Card>
  <Card title="CLI-driven" icon="setting">
    `unextension sync` scaffolds your extension output. `unextension build` compiles it. Simple.
  </Card>
</CardGrid>
